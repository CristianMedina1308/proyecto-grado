📁 ESTRUCTURA DE DATOS GUARDADOS

Para cada ejecución de pruebas:

┌─────────────────────────────────────────────────┐
│ JSON CON:                                       │
│ • Timestamp (fecha/hora exacta)                │
│ • Estado (EXITOSO o FALLÓ)                     │
│ • Versión de PHP                               │
│ • Total de pruebas                             │
│ • Pruebas exitosas                             │
│ • Pruebas fallidas                             │
│ • Pruebas saltadas                             │
│ • Tasa de éxito (porcentaje)                   │
│ • Detalles de cada prueba individual           │
│ • Mensajes de error (si hay)                   │
└─────────────────────────────────────────────────┘

Almacenamiento:
┌────────────────────────────────────────────────────┐
│ reports/latest-report.json       (último)          │
│ reports/logs/report_TIMESTAMP.json (histórico) ... │
└────────────────────────────────────────────────────┘

🔌 API PARA PROGRAMADORES

Acceso HTTP GET desde JavaScript, Python, etc:

┌──────────────────────────────────────────────────────────────┐
│ //Listar todos                                               │
│ GET /reports/api.php?action=list                             │
│                                                              │
│ // Obtener último                                            │
│ GET /reports/api.php?action=latest                           │
│                                                              │
│ // Obtener reporte específico                                │
│ GET /reports/api.php?action=get&file=report_2026-04-23...   │
│                                                              │
│ // Estadísticas generales                                    │
│ GET /reports/api.php?action=stats                            │
└──────────────────────────────────────────────────────────────┘

Respuesta: JSON con estadísticas y detalles de pruebas

❓ FAQ RÁPIDO

P: ¿Dónde se guardan los reportes?
R: En carpeta reports/ del proyecto

P: ¿Se pierden después de ejecutar de nuevo?
R: No, últimose guarda en latest-report.json y histórico en logs/

P: ¿Puedo ver reportes antiguos?
R: Sí, en reports/index.html hay todos los reportes históricos

P: ¿Cómo automatizar ejecutar pruebas cada hora?
R: Usa cron job (Linux) o Task Scheduler (Windows)

P: ¿Puedo usar esto en un pipeline CI/CD?
R: Sí, hay ejemplos en REPORTE_PRUEBAS.md
