❌ PROBLEMA ORIGINAL

  1. Intentas acceder a: localhost/integrador-main/reports/
  2. Pero estás en Railway, no en tu máquina local
  3. Los archivos JSON se pierden en cada deploy
  4. La URL correcta debería ser: proyecto-grado-production-18cd.up.railway.app
  5. Pero no hay forma de guardar persistentemente

                                    ├─ stats

📊 NUEVO FLUJO DE TRABAJO

LOCAL (Desarrollo):
┌───────────────┐      ┌──────────────────────┐      ┌─────────────────────┐
│ Haces cambios │  ->  │ php run-tests-       │  ->  │ Ver reportes en:    │

└───────────────┘      └──────────────────────┘      │ index-db.html       │
                              ↓                      └─────────────────────┘
                         Guarda en BD
                         + archivos JSON


GITHUB:
┌───────────────┐      ┌──────────────────┐      ┌──────────────────────┐
│ git add .     │  ->  │ git commit        │  ->  │ git push             │
│ git commit -m │      │ + mensajes        │      │ (automático)         │
│ "changes"     │      │                  │      │                      │
└───────────────┘      └──────────────────┘      └──────────────────────┘


RAILWAY (Producción):
┌──────────────────────────────┐      ┌─────────────────────────────────┐
│ Recibe push                  │  ->  │ Deploy automático              │
│ Crea contenedor nuevo        │      │ (archivos JSON se pierden,     │
│ (archivos se pierden)        │      │ pero la BD PERSISTE)           │
└──────────────────────────────┘      └─────────────────────────────────┘
                                                 ↓
                                      Ver reportes en:
                                      proyecto-grado-production-18cd.
                                      up.railway.app/reports/index-db.html

⚠️ IMPORTANTE

NO NECESITAS HACER COMMITS AUTOMÁTICOS.

El flujo es:
1. Tú haces cambios
2. Tú haces git commit y git push (tú controlas esto)
3. GitHub recibe el push
4. Railway se dispara automáticamente (automático, sin intervención)
5. Los reportes se guardan en BD (automático)
6. Los datos persisten (no se pierden)
